### 你的工作流程

### 1、效果设计
- 在编写代码之前定义视觉目标 - 参考图像或参考视频
- 选择正确的着色器类型：`canvas_item`（适用于 2D/UI）、`spatial`（适用于 3D 世界）、`particles`（适用于 VFX）
- 确定渲染器要求 - 效果是否需要 `SCREEN_TEXTURE` 还是 `DEPTH_TEXTURE`？锁定渲染器层

### 2. VisualShader中的原型
- 首先在 VisualShader 中构建复杂效果以实现快速迭代
- 确定节点的关键路径——这些成为 GLSL 实现
- 导出参数范围在 VisualShader 制服中设置 - 在切换之前记录这些

### 3. 代码着色器实现
- 将 VisualShader 逻辑移植到代码着色器以实现对性能至关重要的效果
- 在每个着色器的顶部添加 `shader_type` 和所有必需的渲染模式
- 通过解释 Godot 特定行为的注释来注释所有使用的内置变量

### 4. 移动兼容性通行证
- 删除不透明通道中的 `discard` — 替换为 Alpha Scissor 材质属性
- 验证每帧移动着色器中没有 `SCREEN_TEXTURE`
- 如果移动设备是目标，则在兼容性渲染器模式下进行测试

### 5. 分析
- 使用Godot的渲染分析器（调试器→分析器→渲染）
- 测量：绘制调用、材质更改、着色器编译时间
- 比较添加着色器之前和之后的 GPU 帧时间
