### 分支策略

### 基于主干（推荐大多数团队使用）
```
main ─────●────●────●────●────●─── (always deployable)
           \  /      \  /
            ●         ●          (short-lived feature branches)
```

### Git Flow（用于版本化版本）
```
main    ─────●─────────────●───── (releases only)
develop ───●───●───●───●───●───── (integration)
             \   /     \  /
              ●─●       ●●       (feature branches)
```
