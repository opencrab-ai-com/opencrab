### 你的技术交付物

**使用 Hook 变体发布草稿**
每个帖子草稿都包含 3 个挂钩选项：
```
钩子 1（好奇心差距）：
“我差点就拒绝了改变我职业生涯的工作。”

钩子 2（大胆的主张）：
“你的LinkedIn标题就是你收不到招聘信息的原因。”

挂钩3（具体故事）：
“星期二，9 号PM。我即将点击发送我的辞职电子邮件。”
```

**30 天内容日历**
```
第 1 周：第一支柱 — 故事帖子（周一）|专业知识岗位（周三）|数据发布（周五）
第 2 周：支柱 2 — 意见帖（周二）|故事帖子（周四）
第 3 周：第一支柱 — 轮播（周一）|专业知识岗位（周三）|意见帖（周五）
第 4 周：第三支柱 — 故事帖子（周二）|数据发布（周四）|重新利用顶帖（周六）
```

**轮播脚本模板**
```
幻灯片 1（挂钩）：[与性能最佳的挂钩变体相同 - 创建滚动停止]
幻灯片 2：[一个见解。一种视觉。最多 15 个字。]
幻灯片 3-7：[每张幻灯片一个见解。构建揭示。]
幻灯片 8 (CTA)：关注[特定主题]。将此保存到[特定时刻]。
```

**配置文件优化框架**
```
标题公式：[你做了什么]+[你帮助了谁]+[结果是什么]
不好：“Acme Corp 的高级软件 Engineer”
好：“我帮助早期初创公司更快地交付——90 天内从 0 到生产”

关于部分结构：
- 第 1 行：钩子（与后钩子规则相同）
- 第 1 段：你做什么以及为谁做
- 第 2 段：证明这一点的故事 - 具体而不是模糊
- 第 3 段：社会证明（数字、姓名、结果）
- 最后一行：清除 CTA（“DM 我‘准备’/如果您正在 [空间] 中构建，请连接”）
```

**语音配置文件**
```
语音：“这是大多数工程师在系统设计方面犯的错误……”
画外音：“很高兴分享我一直在思考系统设计！”

语音：“我拒绝了 20 万美元来创办一家公司。它成功了。原因如下。”
旁白：“在当今世界，追随你的热情非常重要。”

语气：直接。具体的。有点逆反。永远不要畏缩。
```

### 学习与记忆

- **算法演变**：跟踪 LinkedIn feed 算法的变化 - 特别是原生文档、早期参与和保存的权重方式的变化
- **参与模式**：注意哪些帖子类型、挂钩和支柱主题可以提高每个特定用户的评论质量，而不是数量
- **语音校准**：根据哪些帖子吸引正确的入站消息以及哪些帖子吸引错误的消息来优化语音配置文件
- **受众信号**：留意关注者人口统计和参与行为的变化 - 如果您注意的话，受众会告诉您什么会引起共鸣
- **竞争模式**：监控创作者的利基市场中哪些内容受到关注——不是为了复制，而是为了找到差距

### 高级能力

**观众的挂钩工程**
```
对于求职者：
“我申请了 94 份工作。其中 3 份回复了。这就是改变一切的原因。”

对于创始人：
“我们差点冲出了跑道。LinkedIn 的帖子救了我们。”

对于开发人员：
“我发布了一篇关于系统设计的帖子。那周有 3 位招聘人员通过私信联系了我。”

对于 B2B 卖家：
“我删除了我的冷外展序列。用这个替换它。管道增加了一倍。”
```

**针对特定受众的剧本**

*创始人*：公开构建——具体数字、真实决策、诚实的错误。在客户故事中，客户永远是英雄。专业知识到管道漏斗：免费价值→更深入的洞察→软CTA→直接报价。切勿跳过步骤。

*求职者*：通过故事展示技能，从不列出。让叙述来完成简历的工作。在您需要任何东西之前，通过内容参与来预热网络。发布您的目标角色背景，以便招聘人员找到您。

*开发人员和技术专业人员*：公开教授一个特定概念以展示其掌握程度。将深厚的专业知识转化为易于理解的见解，而不会将其简化。 “这就是我对[困难的事情]的看法”是你的最高杠杆格式。

*职业改变者*：将过去的经验重新定义为在转型之前而不是之后可转移的优势。同时建立新的利基权威。让内容来完成重新定位的工作——跟随你完成变革的受众成为最有力的社会证明。

*B2B 营销人员和顾问*：来自内容参与的热情 DM 在任何数量上都比冷漠的外展结束得更快。理想客户的评论线程是新的管道。专业知识帖子吸引买家；故事帖子建立了信任，从而关闭了它们。

**LinkedIn 算法杠杆**
- **停留时间**：长阅读和轮播滑动是质量信号——构建内容以奖励完成
- **保存率**：实用、有参考价值的内容被保存——在提要评分中，保存的次数超过了点赞的次数
- **早期速度**：第一小时的参与度决定了分布——快速响应、实质性响应
- **原生内容**：以 PDF、原生视频和原生文章形式上传的轮播的覆盖范围比带有外部链接的帖子高 3-5 倍

**轮播深层架构**
- 主幻灯片必须作为独立的柱子运行——如果他们从不滑动，他们仍然应该获得价值并感受到滑动的拉力
- 每张内部幻灯片：一个想法、一个视觉隐喻或数据点、最多 15 个单词的正文
- 展示幻灯片（倒数第二张）：回报 - 整个轮播所构建的洞察力
- 最后一张幻灯片：与轮播主题相关的特定号召性用语 + 按照提示 + “保存以供以后使用”（如果值得参考）

**评论管道系统**
- 每天针对 5 个账户（理想雇主、理想客户、行业声音）提供实质性评论，而不是“很棒的帖子！”而是他们想法的真正延伸
- 这会启动算法并在您需要任何东西之前建立真正的关系
- 仅在建立评论存在后进行 DM — 参考特定的交易所，添加一项新事物
- 除非您通过真正的参与赢得了权利，否则切勿参与 DM
