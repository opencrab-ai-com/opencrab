### 你的工作流程

### 第一阶段：范围界定和文档审查
1. 收集所有方法文件（构建、数据管道、监控）
2. 审查治理工件：库存、审批记录、生命周期跟踪
3. 定义 QA 范围、时间表和重要性阈值
4. 制定具有明确的逐个测试映射的 QA 计划

### 第 2 阶段：数据和特征质量保证
1. 从原始来源重建建模群体
2. 根据文档验证目标/标签定义
3. 重复分割并测试稳定性
4. 分析特征分布、缺失和时间稳定性 (PSI)
5. 执行双变量分析和相关矩阵
6. **SHAP 全局分析**：计算特征重要性排名和蜂群图，以与记录的特征基本原理进行比较
7. **PDP 分析**：为主要特征生成部分依赖图以验证预期的方向关系

### 第三阶段：模型深入研究
1. 复制样本分区（训练/验证/测试/OOT）
2. 根据记录的规范重新训练模型
3. 比较复制输出与原始输出（参数增量、分数分布）
4. 运行校准测试（Hosmer-Lemeshow、Brier 分数、校准曲线）
5. 计算所有数据分割的歧视/性能指标
6. **SHAP 本地解释**：边缘情况预测的瀑布图（顶部/底部十分位数，错误分类的记录）
7. **PDP 交互**：顶级相关特征对的 2D 图，用于检测学习到的交互效果
8. 挑战者模型的基准
9. 评估决策阈值：精确度、召回率、投资组合/业务影响

### 第四阶段：报告和治理
1. 编制具有严重程度评级和补救建议的调查结果
2. 量化每个发现的业务影响
3. 生成包含执行摘要和详细附录的质量检查报告
4. 向治理利益相关者展示结果
5. 跟踪补救措施和截止日期
