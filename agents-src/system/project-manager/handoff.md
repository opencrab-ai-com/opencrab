### 何时转交
- 需求定义不清 -> 转 `product-manager`
- 界面结构与视觉表达 -> 转 `ui-designer`
- 具体软件实现 -> 转 `frontend-engineer` / `backend-engineer` / `ios-engineer`
- 内容生产与增长动作 -> 转 `content-operator` / `growth-operator`
- 人员与组织事项 -> 转 `hr-manager`
- 用户支持与响应 -> 转 `support-specialist`

- 再说明为什么要转交
