# opencrab V1 产品规划

## 1. 产品定位

`opencrab` 是一个面向普通用户的开源 Web 助手产品，底层依赖 Codex SDK。

V1 的目标：
- 主入口是聊天，整体交互参考 ChatGPT Web
- 借鉴 openclaw 的核心能力，但优先追求简单和稳定
- 当前只考虑 Web，运行环境先只考虑 mac
- 产品默认以中文为主，后续再考虑多语言

## 2. 产品边界

### 2.1 要做

- 聊天式 Web 主界面
- 历史对话查看
- 对话文件夹管理
- 图片上传和文件上传
- Codex 支持范围内的模型选择
- 推理强度选择
- Channel 绑定与完整远程对话
- 定时任务的提醒与自动执行
- Skills 的查看、启用和基础管理
- 左下角设置入口和少量低频配置

### 2.2 不做

- 接入其他不同大模型或做多模型路由平台
- 多平台客户端
- 复杂团队协作和权限体系
- 开放式插件市场
- 高自由度工作流编排
- 过多的个性化 UI 配置
- 完整多语言支持

## 3. 设计原则

- 默认简单：普通用户进入产品后应能直接开始聊天
- 原生能力直出：Codex 已具备的能力优先直接使用，不额外重复开发
- 少而稳：第一版不追求功能多，优先把关键闭环做扎实
- 中文优先：界面、提示和默认交互优先面向中文用户
- 设置低频化：设置只保留系统级、低频内容

## 4. UI 参考

V1 可以直接以 ChatGPT Web 为主要 UI 参考对象，优先学习其整体布局、留白、层次和低干扰交互；输入框能力则参考 Codex 桌面应用。

保留的方向：
- 左侧栏 + 主对话区 + 底部大输入框
- 简洁留白的首页空态
- 历史会话在左侧、当前会话在主区
- 左下角设置入口

需要调整的地方：
- 左侧一级入口应突出 `对话`、`Channels`、`任务`、`Skills`
- `对话` 入口下需要支持文件夹和历史对话管理
- 输入框需要支持上传、模型选择、推理强度选择
- 首页空态文案应更贴近中文用户和任务协助场景
- 首页应尽量克制，不做 dashboard 式信息卡片和额外快捷入口

## 5. 信息架构

### 5.1 统一骨架

V1 建议保持单一主框架：
- 左侧栏承担一级导航和列表浏览
- 主区域承担详情、会话和结果展示
- 底部输入框主要服务聊天
- 设置放在左下角

首页状态下，主区域应保持极简，只保留品牌标题、欢迎文案和输入框。

### 5.2 左侧一级入口

- 对话
- Channels
- 任务
- Skills

这四个都是同等重要的核心入口。

### 5.3 对话入口

左侧栏展示：
- 新对话
- 文件夹列表
- 文件夹下的历史对话

第一版建议支持：
- 创建、重命名、删除文件夹
- 将对话移动到文件夹
- 默认的全部对话视图

### 5.4 设置入口

设置位于左下角，只保留低频内容，例如：
- 权限模式
- 默认模型
- 默认推理强度
- 连接状态相关配置
- 账户或应用基础信息

## 6. 页面结构

### 6.1 对话页

`对话` 是默认首页，也是最重要的主入口。

主区域需要支持：
- 默认空态欢迎区
- 当前会话内容
- 执行状态和结果反馈
- 自动任务或远程对话结果回流

输入框需要支持：
- 文本输入
- 图片上传
- 文件上传
- 模型选择
- 推理强度选择

输入框相关约束：
- 模型和推理强度的控制项放在输入框区域，不放在顶部主标题栏
- 首页不额外放置快捷入口、推荐操作或语音输入入口
- 首页视觉上尽量接近 ChatGPT 的极简空态

建议的首页空态文案方向：
- 今天想让 opencrab 帮你做什么？
- 发一句话，或上传文件开始

### 6.2 Channels 页

`Channels` 是一级入口，用于连接和管理远程对话通道。

主区域可包含：
- 当前 Channel 详情
- 连接引导
- 最近远程会话或消息记录
- 与当前 Channel 相关的操作

其定位是“完整远程对话入口”，不是复杂技术配置面板。

### 6.3 任务页

`任务` 是一级入口，用于承载提醒和自动执行任务。

主区域可包含：
- 当前任务详情
- 下次执行时间
- 最近执行记录
- 最近执行结果

第一版至少支持：
- 创建任务
- 查看任务
- 暂停和恢复任务
- 查看执行记录和结果

任务创建应优先使用自然语言或简单表单，不直接暴露复杂规则表达式。

### 6.4 Skills 页

`Skills` 是一级入口，用于展示和管理可用能力。

主区域可包含：
- Skill 简介
- 使用说明
- 启用状态
- 基础操作

第一版只做查看和基础启用管理，不扩展成复杂市场。

## 7. 三条关键闭环

- 聊天闭环：用户打开网页，发起中文对话，调用 Codex 能力完成任务，并在会话中看到结果和历史记录
- 远程对话闭环：用户绑定一个 Channel 后，可以离开网页继续完整远程对话，并在网页中看到相关记录与结果
- 定时任务闭环：用户创建一个定时任务，任务到点后自动执行，并把执行结果回传到网页或 Channel
